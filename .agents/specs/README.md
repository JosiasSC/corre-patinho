# Especificações — corre-patinho

Este diretório contém as especificações autoritativas do projeto.

## Convenções

- Cada spec tem um arquivo completo (`NN-NOME.md`) e um summary (`NN-NOME-summary.md`)
- O summary (~3KB) deve ser lido primeiro; a spec completa é para consulta de seções específicas
- Specs usam Markdown com listas de marcadores, tabelas, blocos de código e linguagem imperativa direta
- O summary DEVE ter YAML frontmatter; o arquivo completo NÃO tem frontmatter
